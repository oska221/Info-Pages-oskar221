Fill a directory with image files and check their integrity.
This tool allows you to simulate disk usage by creating PNG files and later verify their integrity using SHA256 checksums. A simple Tkinter GUI is provided for easy operation.

Project Overview

The program allows you to:

  • Fill a target directory with PNG image files to simulate disk usage.
  • Optionally use a base image file, which will be duplicated for all files.
  • Leave a specified amount of free disk space (default 100 MB).
  • Optionally fill the disk completely (use at your own risk).
  • Verify file integrity using SHA256 checksums stored in checksums.json.
  • Track all operations and verification results in the GUI log.

How It Works

Filling a Directory

  1. Prepare a base image or generate a default PNG.
  2. Check available free disk space.
  3. Create PNG files in the target directory.
  4. Record metadata for each file:
    • SHA256 hash
    • File size
    • Creation timestamp
    • in checksums.json
  5. Stop automatically when the configured free space is reached (or fill to zero if enabled).

Verifying Files

  1. Read checksums.json from the target directory.
  2. Compute SHA256 hashes for all files.
  3. Compare results with the saved hashes.
  4. Log any missing or corrupted files in the GUI.
